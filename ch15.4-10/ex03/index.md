### 15.4-10.1 および 15.4-10.2 の ToDo アプリに対してブラウザの開発者ツールから値の変更やプロパティの追加を試してみなさい

`npm run server`でサーバーを起動し、ブラウザで`http://localhost:3000/ch15.4-10/ex02`にアクセスする。
開発者ツールを開き、Elementsタブで該当箇所の要素の上で右クリックを行い、「Edit as HTML」を選択することで、内容を直接変更したり、追加したりすることができた。
また、Sourceタブで`index.js`を開き、コードを直接変更することもできた。

### 開発者ツールで CSS に関して実行できる操作を検索エンジンで調べ、便利だと思ったものを 3 つ挙げなさい

- CSSの無効化・有効化の切り替え
  ElementタブのStylesセクションで、該当要素のCSSのプロパティのチェックボックスをオン・オフすることで、特定のCSSを無効化・有効化できる。
  そうすることで、スタイルの影響を確認できる。

- 要素の状態（:hover, :active など）の強制適用
  開発者ツールのElementsタブで、該当要素を選択し、右クリックメニューから「Force state」を選択することで、要素の状態を強制的に適用できる。
  これにより、マウスでの操作をシミュレートすることができる。

- レスポンシブデザインモード
  開発者ツールのスマホアイコンをクリックすることで、レスポンシブデザインモードに切り替えることができる。
  これにより、スマホやタブレットなど、異なる画面サイズでの表示を確認できる。

### 15.4-10.2 のアプリの body 要素に対し、元々 HTML および JS 内で利用していなかった Tailwind CSS のクラス (bg-rose-600 など何でも良い) を開発者ツールから追加すると変更が反映されないが、これは何故か調べなさい

Tailwind CSS はビルドプロセスで「ユーティリティクラスを全てあらかじめ CSS に書き出しておく」仕組みを使っているため、その時に定義されているクラウスだけを検出して生成している。

そのため、開発者ツールで動的にクラスを追加しても、Tailwind CSS のビルドプロセスで生成された CSS にはそのクラスが含まれていないため、変更が反映されない。
